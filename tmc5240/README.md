## 概要

Picoから[RPZ-Stepper](https://www.indoorcorgielec.com/products/rpz-stepper/)拡張基板
を使ってステッピングモーターを制御するサンプルプログラムです。


## 対応製品

- [RPZ-Stepper](https://www.indoorcorgielec.com/products/rpz-stepper/)


## 使い方

プロジェクトの開き方など、各サンプルプログラム共通の使い方は[こちら](../)を参照してください。

### シリアルモニター

USBとUART0/GP0の両方へ動作ログを出力します。シリアルモニターでいずれかをモニターすることで、動作ログを確認することができます。

### サンプルプログラムの切り替え

ステッピングモータードライバーの機能が多いため、複数のサンプルプログラムを用意しています。
実行するサンプルプログラムを切り替えるには、
[CMakeLists.txt39行目](CMakeLists.txt#L39)のソースファイルを変更してください。

指定できるソースファイルは以下のとおりです。
| ソースファイル | 内容 |
| ---- | ---- |
| [basic_positioning.cpp](basic_positioning.cpp) | 基本的な使い方&モーターを指定位置まで回転させる |
| [velocity.cpp](velocity.cpp) | モーターを指定速度(回転数)で回転させる |
| [current.cpp](current.cpp) | 電流設定, 電流値とドライバーIC温度測定 |
| [acceleration.cpp](acceleration.cpp) | 加減速を細かく指定する |
| [silent.cpp](silent.cpp) | 静音モード |
| [2motors.cpp](2motors.cpp) | 2つのモーターを制御する |
| [4motors.cpp](4motors.cpp) | 基板2枚で4つのモーターを制御する |


### basic_positioning.cpp

TMC5240Selectorクラスを使ってモーターを制御する基本的な手順を解説しています。
このクラスは、TMC5240モータードライバーを制御する機能と、RPZ-Stepper基板で複数のモーターを制御する機能を有しています。

詳しい動作については[ソースコード](basic_positioning.cpp)内のコメントを参照してください。
ここでは重要な点のみピックアップします。

まず、[18行目](basic_positioning.cpp#L18)でクラスのインスタンスを作成しています。
第1引数の200はモーターの1回転のフルステップ数を指定しており、使用するモーターに合わせて変更してください。
第2引数の0は2つ接続できるモーターのいずれかを選択するもので、ここではJ3端子のモーター#1を指定しています。
~~~
TMC5240Selector m1(200, 0);
~~~

次に、[27行目](basic_positioning.cpp#L27)でモーターに流す最大電流[A]を指定しています。
使用するモーターの定格に合わせて変更してください。
~~~
m1.set_ifs(0.5);
~~~

準備ができたら、J3端子のモーター#1にモーターを、J2端子に電源を接続し、プログラムを実行してください。
モーターが回転すれば成功です!

基本的な制御の仕組みについて解説します。set_xtarget関数で目標の位置を指定すると、
指定した速度([28〜30行目](basic_positioning.cpp#L28-L30))でモータードライバーが自動的にモーターを回転させます。
位置の指定はマイクロステップ単位で行います。一般的な200フルステップで1回転するモーターの場合、
各フルステップが256のマイクロステップに分割されます。そのため、200x256=51200でちょうど1回転することになります。

51200を設定して1回転させた後に0を指定した場合、逆方向に1回転して元の位置に戻ります。

Raspberry Pi + Python向けの解説もWebで公開していますので参考にしてください。
言語は異なりますが、各パラメーターの仕組みなどは同様です。
- [ステッピングモーターを指定位置まで回転させる](https://www.indoorcorgielec.com/resources/raspberry-pi/rpz-stepper-basic-positioning/)


### velocity.cpp

モーターを指定速度(回転数)で回転させるサンプルプログラムです。
詳しい動作については[ソースコード](velocity.cpp)内のコメントを参照してください。
ここでは重要な点のみピックアップします。

[28行目](velocity.cpp#L28)でモーターに流す最大電流[A]を指定しています。
使用するモーターの定格に合わせて変更してください。
~~~
m1.set_ifs(0.5);
~~~

通常、モータードライバーは位置指定モードで動作しています。
この状態ではset_xtargetで指定された目標に向かって回転し、到達すると停止します。

一方、指定した速度で回転させ続ける速度指定モードを搭載しており、set_rampmodeで切り替えが可能です。
~~~
m1.set_rampmode(TMC5240::RAMPMODE_VELOCITY_POSITIVE);
~~~

速度指定モードでは、set_vmax_rpmで指定した回転数になるように自動的に制御します。
加減速はset_amaxに指定された値を使用します。

準備ができたら、J3端子のモーター#1にモーターを、J2端子に電源を接続し、プログラムを実行してください。
モーターが回転すれば成功です!

Raspberry Pi + Python向けの解説もWebで公開していますので参考にしてください。
言語は異なりますが、各パラメーターの仕組みなどは同様です。
- [ステッピングモーターを指定速度/回転数で回転させる](https://www.indoorcorgielec.com/resources/raspberry-pi/rpz-stepper-velocity-mode/)


### current.cpp

モーターの電流を設定して省電力にしたり、電流値とドライバーIC温度測定するサンプルプログラムです。
詳しい動作については[ソースコード](current.cpp)内のコメントを参照してください。
ここでは重要な点のみピックアップします。

[27行目](current.cpp#L27)でモーターに流す最大電流[A]を指定しています。
使用するモーターの定格に合わせて変更してください。
~~~
m1.set_ifs(0.5);
~~~

準備ができたら、J3端子のモーター#1にモーターを、J2端子に電源を接続し、プログラムを実行してください。
シリアルモニターに測定した電流の値が表示されれば成功です!

set_irunで回転中の電流を、set_iholdで停止中の電流を段階的に変化させることができます。
大きなトルクが不要な状況などで電流を下げることで、省電力にして発熱も抑えることができます。
~~~
m1.set_irun(31);
m1.set_ihold(31);
~~~

また、board_current関数で電流値の測定も可能です。なお、測定値は基板全体の電流値となります。
~~~
m1.board_current();
~~~

Raspberry Pi + Python向けの解説もWebで公開していますので参考にしてください。
言語は異なりますが、各パラメーターの仕組みなどは同様です。
- [ステッピングモーターの電流を最適化して省電力化＆発熱対策](https://www.indoorcorgielec.com/resources/raspberry-pi/rpz-stepper-current/)

### 4motors.cpp

2つの基板をスタックして最大4つのモーターを制御するサンプルプログラムです。
詳しい動作については[ソースコード](4motors.cpp)内のコメントを参照してください。
ここでは重要な点のみピックアップします。

基板2枚で動作させる場合は、RPZ-Stepper基板のスイッチを適切に設定しておく必要があります。
両基板のBD_SELスイッチをONにし, 片方のみBD_IDスイッチをONにしてください。
詳しくは製品ページの
[セットアップ](https://www.indoorcorgielec.com/products/rpz-stepper/#setup-hw)をご確認ください。

本サンプルでは、モーター#1とモーター#4を制御していますが、他のコネクターのモーターを使う場合も同様の手順です。

[33行目](4motors.cpp#L33)でモーター#1に流す最大電流[A]を、
[40行目](4motors.cpp#L40)でモーター#4に流す最大電流[A]を
それぞれ指定しています。
使用するモーターの定格に合わせて変更してください。
~~~
m1.set_ifs(0.5);
m4.set_ifs(0.5);
~~~

準備ができたら、BD_ID OFF基板のJ3 MOTOR1端子、およびD_ID ON基板 / J4 MOTOR2端子にモーターを、
J2端子に電源を接続し、プログラムを実行してください。
2つのモーターが回転すれば成功です!

Raspberry Pi + Python向けの解説もWebで公開していますので参考にしてください。
言語は異なりますが、各パラメーターの仕組みなどは同様です。
- [複数のステッピングモーターを制御する](https://www.indoorcorgielec.com/resources/raspberry-pi/rpz-stepper-multiple-motors/)
